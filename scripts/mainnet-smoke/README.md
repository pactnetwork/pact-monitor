# Mainnet 10-Call Smoke Harness

Post-deploy proof that the live Pact Network V1 stack works end-to-end on
**real mainnet**. Modeled on `scripts/smoke-tier2/` but trimmed for cost: 10
calls instead of 50, no surfpool, no local Postgres, no local Pub/Sub.

This harness does **NOT** spin up infrastructure. It assumes the off-chain
stack (market-proxy, settler, indexer) is already running on Cloud Run, the
on-chain program is deployed, and the 5 user-facing endpoints (`helius`,
`birdeye`, `jupiter`, `elfa`, `fal`) are registered with funded coverage
pools.

## Differences from `smoke-tier2`

|                      | smoke-tier2 (surfpool)             | mainnet-smoke                                                                  |
| -------------------- | ---------------------------------- | ------------------------------------------------------------------------------ |
| Validator            | Local surfpool                     | Real mainnet via **Alchemy** RPC                                               |
| USDC mint            | Test mint generated by harness     | `USDC_MINT_MAINNET` (`EPjFW…`)                                                 |
| Agent funding        | Surfpool cheatcode                 | Pre-funded via real USDC transfers — driver assumes; doesn't try to mint       |
| Pub/Sub              | gcloud emulator                    | Real GCP Pub/Sub on `pact-network-prod`                                        |
| Postgres             | Local docker                       | Cloud SQL via private IP — driver doesn't connect; queries indexer's HTTP API |
| Calls                | 50                                 | 10 (cost — each call burns real Alchemy credits + premium)                     |
| Mix                  | 70/10/10/5/5                       | 7 ok / 1 latency / 1 server / 1 network (skip `client_error`)                  |
| Test agents          | 5 mock                             | 1 dedicated mainnet test agent funded with ~$1 USDC by Rick                    |

> **`helius` slug note.** That's a *user-facing* slug for the Helius RPC API
> we insure on behalf of agents. It's separate from which RPC provider Pact
> itself uses internally for sending settle_batch txs (Alchemy).

## Pre-requisites

1. **Off-chain stack live on Cloud Run.**
   - `market.pactnetwork.io` (proxy) — `/health` returns 200.
   - `indexer.pactnetwork.io` — `/api/stats` returns 200; `/api/endpoints`
     lists all 5 slugs.
   - Settler running, draining the production Pub/Sub topic.
2. **On-chain program deployed.** ProtocolConfig + Treasury +
   SettlementAuthority + 5 EndpointConfig PDAs all initialized.
3. **5 coverage pools funded.** `topup_coverage_pool` already run for each.
   `current_balance > 0` per slug.
4. **Test agent.** Dedicated mainnet keypair with at least 1 USDC + ~0.001 SOL
   (no settler txs are signed by this key — the SOL is just rent if the ATA
   doesn't exist yet, which it shouldn't if you've already done step 5).
5. **`spl-token approve` already run** on the test agent ATA, delegating to
   the on-chain SettlementAuthority signer pubkey. Without this, every
   `settle_batch` referencing this agent fails with a token-program error.
   Example:
   ```
   spl-token approve <ATA> 1000000 <SETTLEMENT_AUTHORITY_SIGNER> --owner <test-agent-keypair.json>
   ```
6. **Test agent in the demo allowlist** (so `?demo_breach=1` actually forces
   a latency_breach). Add via the ops API or the proxy's allowlist Postgres
   table, depending on how the production proxy is wired.

## Environment

Every variable is required (no defaults — we never want to silently target
the wrong cluster). Set in `.env` or your shell before running:

```
MAINNET_RPC_URL=https://solana-mainnet.g.alchemy.com/v2/<your-key>
INDEXER_URL=https://indexer.pactnetwork.io
MARKET_PROXY_URL=https://market.pactnetwork.io
TEST_AGENT_KEYPAIR_PATH=~/pact-mainnet-keys/smoke-test-agent.json
# optional:
# MAINNET_PROGRAM_ID=<override>
# USDC_MINT=<override>
# SETTLER_DRAIN_MS=60000
```

## Run

```bash
cd scripts/mainnet-smoke
pnpm install                # workspace member; pnpm -w install also works
pnpm preflight              # 00-preflight.ts — read-only checks; no USDC burned
pnpm fire                   # 01-fire-10-calls.ts — fires 10 real calls + waits 60s
pnpm reconcile              # 02-reconcile.ts — diffs on-chain vs indexer
```

If preflight fails, fix the reported items and re-run. `01-fire` writes its
result to `.smoke-state/state.json`, which `02-reconcile` reads.

## Pass / fail criteria

A run is **PASS** only when:

1. Preflight reports all checks PASS.
2. Fire returns a valid `callId` for all 10 calls (X-Pact-Call-Id header
   was attached by wrap, meaning the call was insurable and made it through
   classification).
3. For every of the 5 endpoint slugs, on-chain `CoveragePool.current_balance`
   equals indexer `pool.currentBalanceLamports`. Mainnet pools have no seed
   amount to back out (unlike smoke-tier2) — the diff must be exactly zero.
4. For every of the 10 callIds, `GET /api/calls/:id` returns 200 with:
   - `endpointSlug` matching the slug we hit;
   - `breach == false` for `ok` outcomes, `true` otherwise;
   - `breachReason == null` for `ok`, otherwise matching the expected
     classifier outcome string (`latency_breach`, `server_error`,
     `network_error`).

A run is **FAIL** if any of those checks miss. The reconciler prints a
per-row breakdown so you can investigate without grepping logs.

## Cost estimate

- USDC: ~$1 maximum, dominated by 7 successful calls debiting the per-slug
  flat premium. With current premiums (`fal=$0.05`, `elfa=$0.005`,
  `birdeye=$0.002`, `jupiter=$0.0015`, `helius=$0.001`), 7 calls spread across
  the 5 slugs costs about $0.07 in premium. Refunds (latency/server/network)
  return the principal + premium back to the agent, so they're net-zero
  modulo the imputed cost from the pool.
- SOL: ~0.001 SOL for ATA rent if the test agent's USDC ATA doesn't exist
  yet; zero if it already does.
- Alchemy credits: ~30 RPC calls during the run (preflight reads, reconcile
  reads, settler txs from the production stack).

**Per run total: ~$1 USDC + ~0.001 SOL.** Cheap enough to re-run after every
mainnet redeploy.

## Known caveats

- **Server / network error slots are best-effort.** The harness hits
  `/__pact_smoke_5xx` (server_error) and `/__pact_smoke_unreachable`
  (network_error) on the upstream. If those paths return non-error status
  codes from the real upstream, the wrap classifier will assign a different
  outcome and the per-call check will fail. Operators can either:
  - Wire a `/__pact_smoke_5xx` route on a chosen upstream that returns a
    deterministic 5xx, or
  - Accept the deviation and treat the per-call mismatch as expected for
    those two slots, gating only on the 7 `ok` + 1 `latency_breach` outcomes.
- **No teardown.** This harness assumes the production stack stays running.
  If you need to reset between runs, just delete `.smoke-state/`.

## State files

- `.smoke-state/state.json` — fired call list, timestamps. **Gitignored.**

## What this harness does NOT cover

- Re-deploys, fee-recipient rotations, pause / unpause round-trips. Those
  are exercised by `scripts/mainnet/pause-protocol.ts` and the ops API.
- Multi-agent contention. Smoke-tier2 fires from 5 agents in parallel; this
  fires from 1 agent serially. Multi-agent contention is left to load tests.
