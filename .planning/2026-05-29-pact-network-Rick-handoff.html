3. EVM settler bug found & fixed — PR #246 (merged)

Bug: EvmAdapter.submitSettleBatch passed a bare address string as viem's account, so viem emitted wallet_sendTransaction (node-signed) — which Arc/base RPCs reject ("this request method is not supported"). The settler's EVM auto-flush had never actually submitted a tx on any EVM chain (prior Arc settles were done by a manual script).

Fix: pass the local PrivateKeyAccount object so viem signs locally → eth_sendRawTransaction. Surgical (1 functional line + comment + regression test). Both EVM settle txs above are eip1559 raw transactions — the exact behavior the fix introduces. commit a8afaea · PR #246 merged into feat/multi-network.
