3. Bug found & fixed during testing — PR #246 (merged)

Driving the EVM chains to a real on-chain settle surfaced a genuine merge-blocker in the multi-network feature: the settler's EVM auto-flush had never actually submitted a transaction on any EVM chain.

Root cause

EvmAdapter.submitSettleBatch (packages/shared/src/adapters/evm/index.ts) passed account: account.address — a bare address string — to viem's sendTransaction. viem then treats it as a node-managed account and emits wallet_sendTransaction, asking the RPC to sign. Public testnet RPCs reject this — Arc replies literally "this request method is not supported".

Fix

Pass the local PrivateKeyAccount object (the key the wallet client already holds) so viem signs locally and submits via eth_sendRawTransaction. Surgical: 1 functional line + comment, plus a regression unit test. Both settle txs above are eip1559 raw transactions — the exact behavior the fix introduces.

Branch fix/evm-settle-send-raw-transaction · commit a8afaea · PR #246 merged into feat/multi-network. Build clean; @pact-network/shared tests 53/53 (incl. new regression). Blast radius: LOW (internal to EvmAdapter; Solana path untouched).
